# Contributing

## How to Contribute

Contributions to CIMP are welcome. This includes:

- Documentation improvements
- Practice examples
- Bug reports
- Suggestions for improvements

## Contribution Process

1. Review existing documentation and practices
2. Open an issue to discuss significant changes
3. Submit a pull request with your changes
4. Participate in review discussions

## Guidelines

- Follow existing documentation style
- Keep changes focused and well-documented
- Ensure all examples are concrete and actionable
- Maintain the infrastructure-first, governance-oriented tone

## Canon vs Practices

- **Canon changes** require governance review (see [GOVERNANCE.md](../docs/GOVERNANCE.md))
- **Practice changes** can evolve more freely
- **Examples** are welcome additions

## Questions?

Open an issue or contact the maintainers.
